2.9  Research Synthesis: Identified Gaps and Study Contribution
─────────────────────────────────────────────────────────────────────────────────

The preceding sections have reviewed five distinct but converging bodies of
literature.  This section synthesises the identified gaps across these bodies
and articulates the specific contribution the present research makes to the field.

Gap 1: The Semantic Detection Ceiling of Rule-Based Tools.  Section 2.4
established that automated DOM-based accessibility checkers (axe-core, WAVE,
Lighthouse, Pa11y) have a fundamental inability to evaluate the semantic adequacy
of ARIA attributes, accessible names, and dynamic state management.  The
theoretical and empirical evidence reviewed by Abou-Zahra et al. [17] and
Campoverde-Molina et al. [18] quantifies this ceiling at 25-40 per cent of
violations detectable by human experts.  The ground-truth validation conducted
in the present study (1.2% pattern-match accuracy for semantic violations)
confirms and sharpens this finding.  No prior study was identified that addresses
this ceiling through integration of a large language model into the IDE editing
workflow.

Gap 2: The Absence of WCAG 2.2-Aware IDE Tooling.  The WCAG 2.2 specification,
published in October 2023, introduced nine new success criteria addressing
previously uncovered interaction patterns (focus appearance, dragging movements,
target size, accessible authentication).  At the time of writing, no IDE plugin
or integrated developer tool was identified in this review that provides
real-time analysis specifically covering the WCAG 2.2 Level A and AA success
criteria for all four principal web development languages (HTML, CSS, JavaScript,
and TypeScript with JSX).  Existing IDE accessibility plugins confine their scope
to HTML and to the axe-core rule set, an architecture inherited from browser-
based tooling that cannot be straightforwardly extended to CSS logic or JavaScript
state-management analysis.

Gap 3: Missing RAG Application to Accessibility.  Section 2.6 demonstrated
that RAG substantially improves LLM factual accuracy on knowledge-intensive tasks
by grounding model outputs in authoritative external documentation [25].
No prior published work was identified that applies the RAG architecture to
accessibility analysis, either in an IDE-embedded tool or in a benchmarking
study.  The selection, construction, and evaluation of a domain-specific WCAG 2.2
knowledge base for hybrid retrieval therefore represents a novel contribution.

Gap 4: Absence of Multi-Model Comparative Benchmarking for Accessibility.
No prior study was identified that benchmarks a large cohort of diverse LLM
families on a standardised, ground-truth-validated accessibility detection task.
Studies on LLM code-analysis capabilities have focused on code generation
(Codex, HumanEval), bug detection, or general code review, none of which
exercises the WCAG-specific semantic reasoning required for accessibility auditing.
The present study's 18-model, 4-condition, 41-fixture evaluation provides the
first comparative benchmark of this scope for LLM-based WCAG 2.2 detection.

Gap 5: Absence of Cognitive-Load Evidence for Accessibility Tooling.  Section
2.8 reviewed the NASA-TLX instrument and the developer cognitive-load literature.
No prior study was identified that measures the cognitive workload cost of a
web developer accessibility audit task with and without AI assistance, nor that
correlates an AI tool's detection accuracy with developer workload reduction.  The
planned user study described in Chapter 3 (Section 3.4.4) is designed to address
this gap directly, providing the first empirical evidence linking accessibility
AI tool accuracy to NASA-TLX workload scores in a controlled developer task.

The present research addresses all five identified gaps through a single,
integrated study.  The artefact (the VS Code extension) addresses Gaps 1, 2,
and 3 simultaneously.  The evaluation framework (the 18-model ablation study)
addresses Gap 4.  The planned empirical user study (Section 3.4.4) addresses
Gap 5.

─────────────────────────────────────────────────────────────────────────────────

2.10  Chapter Conclusion
─────────────────────────────────────────────────────────────────────────────────

This chapter has reviewed the bodies of literature that motivate, enable, and
contextualise the present research.  The review began with the empirical evidence
for the persistence of web accessibility barriers at scale, a global problem
affecting 1.3 billion people that existing automated tooling has demonstrably
failed to solve, and that legislation in the EU, UK, and US is increasingly
mandating organisations to address.  It then surveyed WCAG 2.2 and WAI-ARIA as
the technical standards that define the problem space; examined the current
generation of automated evaluation tools and characterised their detection
ceiling; and established the semantic accessibility gap that motivates the turn
to language model-based analysis.

The review then traced the development of large language models from the
transformer architecture through GPT-3's few-shot capabilities and Codex's
code intelligence to the emerging application of LLMs to accessibility auditing.
The RAG architecture was reviewed as the mechanism for addressing the hallucination
and knowledge-currency limitations of purely parametric models.  Hybrid BM25
and dense-semantic retrieval was identified as the technically appropriate
retrieval design for the structured WCAG knowledge base.

Prompt engineering research, particularly chain-of-thought prompting and its
zero-shot variant, was reviewed as the technique for eliciting structured,
confidence-gated analytical outputs from the model.  The inference sampling
parameters governing response quality were reviewed with reference to their
empirical foundations, providing the theoretical basis for the parameter choices
made in Chapter 3.

Finally, this chapter reviewed cognitive load theory, the NASA-TLX instrument,
and the developer productivity literature to establish the theoretical and
measurement framework for the human-factors evaluation strand of the study.
The Matthews Correlation Coefficient was reviewed as the preferred primary metric
for binary classification under class imbalance, providing the rationale for its
co-primary role alongside F1 in the benchmarking evaluation.

The synthesis in Section 2.9 identified five distinct gaps across the reviewed
literature, all of which the present research is designed to address, and
articulated the novel contribution of the study at each gap.  Chapter 3 proceeds
to describe the methodology and implementation through which these contributions
are realised.
