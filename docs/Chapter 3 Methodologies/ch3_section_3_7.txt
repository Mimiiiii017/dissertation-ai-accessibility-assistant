3.7  Chapter Summary
─────────────────────────────────────────────────────────────────────────────────

This chapter has presented the complete methodology and implementation of the
AI-assisted accessibility audit extension that constitutes the technical
artefact of this dissertation.  The research objective — to determine whether a
domain-grounded LLM can reliably detect WCAG 2.2 accessibility violations in
real-world web source files and whether improved detection accuracy reduces
developer cognitive workload — was operationalised through a three-layer system
comprising a Visual Studio Code extension, a hybrid RAG service, and an LLM
inference relay.

Section 3.1 contextualised the research within the established literature on
web accessibility prevalence, shift-left development practices, and retrieval-
augmented language models, and formally stated the two principal hypotheses
(H1 and H2).  Section 3.2 defined the domain scope with reference to the WCAG
2.2 Level A and AA success criteria across four file languages; enumerated the
full set of experimental variables including independent, dependent, controlled,
confounding, mediating, moderating, and extraneous variables; and specified all
inference hyperparameters with their literature justifications.

Section 3.3 described the technical implementation in detail, covering the
construction of the dual-collection knowledge base, the hybrid BM25 and dense-
semantic RAG pipeline, the core analysis algorithm, and the two-phase prompt
engineering approach.  Five pre-testing shortcomings were identified —
including a prompt rule contradiction that suppressed accuracy by over 34 F1
percentage points — and seven code refinements were implemented and validated
before the formal benchmarking programme commenced.

Section 3.4 described the pilot study in which UI/UX friction and hallucination
patterns were identified through informal observation, and documented the
subsequent prompt and interface refinements.  The main benchmarking protocol was
specified in full as a reproducible six-step procedure, and the planned user
study design was described including the counterbalanced A/B protocol,
participant selection criteria, and data collection instruments.

Section 3.5 presented the evaluation framework: the 2×2 ablation testing matrix
producing results across 18 models and 4 conditions; the full suite of technical
performance metrics including F1, MCC, and six novel evaluation techniques;
the ground-truth construction and cross-validation procedure; the dual roles of
the NASA-TLX instrument as both a computational prediction model and a user-study
outcome measure; and the statistical correlation plan linking technical accuracy
to subjective cognitive workload.

Section 3.6 addressed ethical considerations, detailing the GDPR-compliant data
management protocol, the pseudonymisation and retention procedures, and the
institutional ethics approval process.

Together, the components described in this chapter provide a complete and
reproducible research platform for evaluating the accessibility detection
capability of retrieval-augmented language models in a real developer workflow
context.  The ablation results confirm the separable positive contributions of
both the RAG condition and the chain-of-thought reasoning condition, with gpt-
oss:120b under RAG + Think achieving the highest composite score of 72.0% — the
configuration deployed in the production extension.  The following chapter
presents the full quantitative results of the benchmarking programme and the
statistical analysis of model performance across fixtures, error densities, and
the adversarial test set.
